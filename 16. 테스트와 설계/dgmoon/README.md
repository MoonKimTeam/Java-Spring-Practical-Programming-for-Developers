# ch.16 테스트와 설계

- 테스트와 SOLID의 상관관계

## 테스트와 SRP

- 단일 책임 관점에서 User 컴포넌트는 분리하는 것이 맞음
    - 컴포넌트 사용 주체가 미가입자와 가입자로 서로 다른 메시지를 보내는 두명의 액터가 볼 수 있기 때문
- 테스트는 계속해서 분리하면 어떨까 신호를 보냄

## 테스트와 ISP

- 테스트는 인터페이스 분리를 유도함
- Dummy 인터페이스를 만들고 나니 불필요한 메서드 구현이 필요해짐
    - 따라서 테스트가 인터페이스를 분리하라는 신호를 보내고 있는 것

## 테스트와 OCP, DIP

- 좋은 설계를 갖춘 시스템은 유연함
- 유연한 코드를 만들었는지 확인하는 방법 -> 컴포넌트간 의존 관계를 전부 확인하기
    - 하지만, 이 방법은 공수가 너무 큼
    - 좋은 대안은 바로 테스트 -> 테스트로 유연성과 확장성이 어떤지 판단 가능

## 테스트와 LSP

- 추상 클래스 상속 및 파생 클래스 테스트를 실행하면서 리스코프 치환을 함께 검증할 수 있음
