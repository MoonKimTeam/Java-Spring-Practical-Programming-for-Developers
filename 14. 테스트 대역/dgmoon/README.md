# ch.14 테스트 대역

- 실제 객체를 대신해서 해동하고 실제 객체가 하지 못하는일을 대신함

## Dummy

- 더미의 역할은 아무런 동작도 하지 않고, 오롯이 코드가 정상적으로 돌아가게 하는 역할

## Stub

- 미리 준비된 값을 반환하는 대역 객체

## Fake

- 테스트를 위한 자체적인 논리를 갖고 있음(더미, 스텁보다 한 단계 더 발전된 유형)

## Mock

- 메서드 호출이 발생 했는지 여부를 검증하는 역할

### 상태 기반 검증

- 테스트의 검증 동작에 상태를 사용하는 것

### 행위 기반 검증

- 테스트의 검증동작에 메서드 호출 여부를 보게하는 것

### 상태 기반 vs. 행위 기반

- 상호 작용 테스트란 행위기납 검증을 하는 테스트
- 가급적 테스트는 상태 기반 검증으로 작성하는 편이 좋음
    - 그렇게 해야 테스트를 책임 단위로 바라볼 수 있게 됨

## Spy

- 만약 실제 개게 였다면 어떤 메서드가 호출되고 이벤트가 발생했는지 등을 기록하고 감시
- 메서드가 몇 번 호출됐는지, 메서드는 어떤 매개변수로 호출됐늕, 메서드 후출 순서는 어떤지 등 모든 것을 기록
